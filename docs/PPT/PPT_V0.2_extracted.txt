PAGE 15:
프로젝트 목표 및 설계-AI Agent 시스템 설계 02
1. 시스템 목적
13연결의  민족
1.1 배경
사용자는 논문 검색, 요약, 용어 설명 등 다양한 요구사항을 가지고 있음
매번 기능을 수동으로 선택하는 것은 비효율적
질문 의도를 자동으로 파악하고 적절한 도구를 실행하는 지능형 시스템 필요
1.2 목표
자동 의도 파악: LLM 기반 라우터로 질문 유형 자동 분석
최적 도구 선택: 7가지 도구 중 가장 적합한 도구 자동 실행
난이도 맞춤 응답: 초심자/전문가 수준에 맞는 답변 제공
다중 작업 처리: "찾아서 요약해줘" 같은 복합 요청 자동 처리
대화 연속성: 이전 대화를 기억하여 맥락 유지2. 기술 스택
프레임워크: LangGraph StateGraph
LLM: OpenAI GPT-5 (temperature=0), Solar Pro2
상태 관리: TypedDict 기반 AgentState
메모리: ChatMemoryManager (멀티턴 대화)
로깅: ExperimentManager (전체 프로세스 추적)
3. 핵심 기술
라우터 노드 (Router Node)
역할: 질문 분석 및 도구 선택
구현 로직:
1.사용자 질문을 LLM에 전달
2.7가지 도구 설명과 함께 프롬프트 생성
3.LLM이 가장 적합한 도구 이름 반환
4.state["tool_choice"]에 저장
