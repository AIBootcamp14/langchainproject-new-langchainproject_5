# 시나리오: 일반 답변 (General Answer)

## 📋 도구 설명

**도구명**: `general`
**목적**: 논문 검색이나 용어집 조회 없이 LLM의 일반 지식으로 답변하는 도구

### 주요 기능
- LLM의 사전 학습된 지식 활용
- 빠른 응답 속도
- 간단한 질문이나 일반적인 AI/ML 개념 설명

---

## 🎯 사용 시나리오

### 시나리오 1: AI 개념 설명 요청

**사용자 상황**: AI/ML 초보자가 기본 개념을 이해하고 싶어함

**사용자 질문**:
```
"머신러닝이 뭐야?"
```

**Agent 동작**:
1. Router가 질문 분석
2. 간단한 개념 질문으로 판단
3. `general` 도구 선택
4. LLM이 사전 지식으로 답변 생성

**예상 답변** (Easy 모드):
```
머신러닝은 컴퓨터가 데이터로부터 스스로 학습하여 문제를 해결하는 기술입니다.

예를 들어:
- 스팸 메일 필터: 수많은 스팸 메일 사례를 학습하여 새로운 스팸을 자동으로 분류
- 음성 인식: 다양한 사람들의 음성 데이터를 학습하여 말을 텍스트로 변환
- 추천 시스템: 사용자의 과거 행동을 학습하여 선호할 만한 상품 추천

사람이 일일이 규칙을 정하지 않아도, 데이터에서 패턴을 찾아 스스로 개선됩니다.
```

**예상 답변** (Hard 모드):
```
머신러닝(Machine Learning)은 명시적으로 프로그래밍하지 않고 데이터로부터 학습하여
성능을 개선하는 알고리즘과 모델의 집합입니다.

주요 학습 패러다임:
1. 지도 학습 (Supervised Learning): 레이블된 데이터로 학습
   - 분류 (Classification): 이산적 클래스 예측
   - 회귀 (Regression): 연속적 값 예측

2. 비지도 학습 (Unsupervised Learning): 레이블 없는 데이터에서 패턴 발견
   - 군집화 (Clustering)
   - 차원 축소 (Dimensionality Reduction)

3. 강화 학습 (Reinforcement Learning): 환경과의 상호작용을 통한 학습

수학적 기초: 최적화 이론, 확률론, 선형대수를 활용하여 손실 함수를 최소화하는
파라미터를 찾습니다.
```

### 시나리오 2: 비교 질문

**사용자 질문**:
```
"딥러닝과 머신러닝의 차이는 뭐야?"
```

**Agent 동작**:
1. 비교 질문 → DB 검색 불필요
2. `general` 도구 선택
3. LLM의 지식으로 비교 설명

**예상 답변**:
```
머신러닝과 딥러닝의 관계:
- 딥러닝은 머신러닝의 하위 분야입니다

주요 차이점:
1. 모델 구조:
   - 머신러닝: 의사결정나무, SVM, 랜덤 포레스트 등
   - 딥러닝: 다층 신경망 (Deep Neural Networks)

2. 특징 추출:
   - 머신러닝: 사람이 수동으로 특징(feature) 설계
   - 딥러닝: 모델이 자동으로 특징 학습

3. 데이터 요구량:
   - 머신러닝: 상대적으로 적은 데이터로 학습 가능
   - 딥러닝: 대량의 데이터 필요

4. 성능:
   - 작은 데이터: 전통적 머신러닝이 유리
   - 대용량 데이터: 딥러닝이 더 높은 성능
```

### 시나리오 3: 일반 대화

**사용자 질문**:
```
"안녕?"
```

**Agent 동작**:
1. 인사말 인식
2. `general` 도구 선택
3. 자연스러운 응답

**예상 답변**:
```
안녕하세요! 👋

저는 논문 리뷰 챗봇입니다. AI/ML 관련 논문 검색, 용어 설명, 웹 검색 등을 도와드릴 수 있습니다.

어떤 도움이 필요하신가요?
```

## 📊 성능 지표

- **응답 속도**: 빠름 (DB 검색 없음)
- **정확도**: LLM의 사전 지식에 의존
- **비용**: 낮음 (단일 LLM 호출)

---

## ⚠️ 제한사항

1. **최신 정보 부족**: LLM 학습 시점 이후의 최신 논문/기술은 모름
2. **논문 인용 불가**: 구체적인 논문 출처 제공 불가
3. **환각(Hallucination) 가능**: 잘못된 정보 생성 가능성

## 🔄 다른 도구로의 전환

다음과 같은 경우 다른 도구가 더 적합합니다:

| 상황 | 추천 도구 |
|------|----------|

| 논문 요약 요청 | `summarize` (논문 요약) |
| 답변 저장 필요 | `save_file` (파일 저장) |
